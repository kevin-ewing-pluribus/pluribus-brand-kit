Pluribus Brand Guidelines

Canonical identity system for anything generated by Pluribus. Use by default unless explicitly overridden.

Version 1.1.0 • Palette aligned to kevin-ewing.com

1) Brand Intent

Positioning: Calm, competent operator. High-signal output. Minimal fluff.

Personality: Precise, direct, useful, professional.

Recognition goal: You can instantly identify “this came from Pluribus.”

2) Logo and Favicon

Primary logo: pluribus-logo.svg

Favicon: pluribus-favicon.svg

Usage rules

  • Minimum digital size: 24px icon / 96px full lockup.
  • Clear space: at least half the icon width around logo.
  • Do not recolor logo outside approved palette.

3) Color System

Base Background #191818
Surface #1F1F1F
Primary #0074F0
Secondary #14A9FF
Success #32A94C
Danger #BF2626

Primary gradient: linear-gradient(135deg, #0074F0 0%, #14A9FF 100%)

Accessibility baseline: body text contrast >= WCAG AA.

4) Typography

UI Sans: Inter, SF Pro Text, system sans fallback.

Monospace: JetBrains Mono, SFMono fallback.

Scale: H1 34 / H2 22 / H3 16 / Body 15 / Meta 12

Text rhythm: line-height 1.45-1.6, consistent spacing blocks.

5) Shapes, Spacing, Motion

Shape language

Rounded rectangles, soft corners, no sharp ornamental geometry.

radius-sm 8 radius-md 12 radius-lg 16

Spacing system

4, 8, 12, 16, 24, 32, 48px increments only.

Dense info blocks use 12-16px vertical rhythm.

6) Communication Signature

  • Lead with result, then concrete facts.
  • Metrics must be defined in plain language.
  • Avoid abstract KPI labels without definitions.
  • One clear “Top Next Move” recommendation.

Canonical sign-off

— Pluribus

7) Component Defaults

Email reports (default)

  • Mobile-first single-column layout.
  • Section order: System Health, Automation Status, Change Report, Tool Discovery, Top Next Move.
  • No unexplained percentages.

Status chips

Healthy Watch Issue

8) Do / Don’t

Do

  • Use the Pluribus logo and palette consistently.
  • Keep outputs crisp and interpretable.
  • Use token files for transferability across products.

Don’t

  • Use random colors/fonts per message.
  • Ship mystery metrics with no definitions.
  • Over-decorate at expense of readability.

9) Transferability Pack

This kit is portable across web apps, email templates, docs, dashboards.

  • tokens.css (CSS variables)
  • tokens.json (design-token interchange)
  • pluribus-logo.svg
  • pluribus-favicon.svg
